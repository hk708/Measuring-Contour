Algorithm Overview

First we create a set of iso rectangles which are enclosed under one frame, which we then split into two halves such that both halves have roughly the same number of vertical edges. Now, when we come across a frame that contains a singular edge, we must then again split this frame into 3 parts or stripes. The first stripe along the edges of the rectangle, the second stripe being above the edges of the rectangle and the final one being below the edges of the rectangle. If the vertical edge we come across is a left edge, then the area on the other side of the frame and in the middle, will lie inside the set of iso rectangles. Now, while merging multiple frames, we start off with the introduction of the stripes from each frame to the other (We make more partitions along the Y axis in the respective frame). Now for each given stripe of a frame, if the edge contained does not have a partner edge in the merged frame, then the relevant stripe of the merged frame will definitely be enclosed in our set of iso rectangles. We repeat this merging process until the original frame is obtained. The contour pieces in this frame were formed by the parts of horizontal edges, and the adjacent stripe in which the rectangles are not enclosed, is also not covered.

Time Complexity of the Algorithm

We assume T(n) to be the complexity of the STRIPES algorithm. The problem is divided into 2 subproblems of roughly the same size in the divide step. Since the base case works on only one edge, it takes constant time to run. Since the set operations take linear time, we can conclude that the merge step also runs in linear time, which gives us:


T(n) = 2T(n/2) + O(n)


When we solve this, we get:


T(n) = O(nlog(n))
